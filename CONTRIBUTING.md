# Contributing to Universal Manifest

Universal Manifest is an open specification. Contributions to the spec are welcome.

## Providing Feedback

Open a [GitHub Issue](https://github.com/UniversalManifest/universal-manifest/issues) describing what you found, what you expected, and which spec version is affected.

## Proposing Specification Changes

Specification changes go through a structured process:

1. **Open an issue** describing the problem and your proposed change. Include which section of the spec is affected.
2. **Discussion** happens on the issue. The maintainers will evaluate the proposal against the current spec direction and existing implementations.
3. If accepted, **submit a pull request** with the change. Spec changes must be surgical: modify only the affected sections. A diff between your PR and the current spec should show exactly what changed and nothing else.

Do not submit a PR without an issue first. Undiscussed changes will be closed.

## What This Repo Contains

This repository contains the Universal Manifest specification and supporting schema files. It does not contain SDKs, reference implementations, or tooling. Contributions should be limited to the specification itself.

Please read [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) before participating.
